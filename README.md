# n8n-nodes-bambulab

Custom n8n nodes for local printer monitoring and control, built for use with [Bambu Lab](https://bambulab.com) printers over the local network.

## Nodes

### Bambu Lab Printer

Interact with a Bambu Lab printer directly over your local network using MQTT — no cloud required.

**Operations:**

| Operation    | Description                                                                   |
| ------------ | ----------------------------------------------------------------------------- |
| Get Status   | Request a full status push from the printer and return the raw report payload |
| Send Command | Publish a raw JSON command to the printer and return the first response       |

## Setup

### 1. Install the package

Load this repository as a custom extension in your n8n instance by setting the environment variable:

```
N8N_CUSTOM_EXTENSIONS=/path/to/this/repo
```

If using the included `docker-compose.yml`, this is already configured.

### 2. Create a credential

In the n8n UI go to **Credentials → New** and search for **Bambu Lab (Local Network)**.

You will need three values from your printer:

| Field                     | Where to find it                                                                                                                               |
| ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **Printer IP Address**    | Printer touchscreen → Settings → Network / WLAN                                                                                                |
| **Printer Serial Number** | Printer touchscreen → Settings → Device — the long alphanumeric ID (e.g. `A1B2C3D4E5F6A1B2`), not the `3DP-...` model name. Must be uppercase. |
| **Access Code**           | Printer touchscreen → Settings → Network / WLAN — 8-character code generated by the printer                                                    |

When prompted for a name, enter **"Bambu Lab Local"** — the showcase workflow references this exact name.

### 3. Import the showcase workflow

A ready-made workflow is included at `workflows/bambulab-showcase.workflow.json`.

Import it via the n8n CLI:

```bash
docker compose exec n8n n8n import:workflow \
  --input=/custom-extensions/my-custom-node/workflows/bambulab-showcase.workflow.json
```

Or import it manually from the n8n UI via **Workflows → Import from file**.

> **After importing:** open the workflow, click the **Get Printer Status** node, select your **"Bambu Lab Local"** credential from the dropdown, and save. This links the credential ID to the workflow.

The showcase workflow contains:

- **Get Printer Status** — fetches the full MQTT report and passes it to the next node
- **Extract Key Fields** — flattens the raw payload into a clean object with temperatures, print progress, state, job name, and error code
- **Send Raw Command** _(disabled by default)_ — example of sending a pause command

## Running n8n locally

### Build

```bash
npm install
npm run build
```

### Start

```bash
docker compose up -d
```

n8n will be available at http://localhost:5678.

### Rebuild after changes

```bash
npm run build
docker compose restart n8n
```

### Check running vs latest n8n version

```bash
npm run check:n8n-version
```

## Security notes

- Credentials are stored encrypted in n8n's internal database and are never written to disk or committed to git.
- The MQTT connection to the printer uses TLS on port 8883. Certificate verification is disabled because Bambu printers use a private CA — the connection is still fully encrypted.
- Do not expose port 8883 or your printer's IP to the public internet.
